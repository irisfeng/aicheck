# 2026-06-09

## Today

- Diagnosed an operator-facing `Failed to fetch` error that appeared after clicking 开始分析 on the production site.
- Root cause: the production domain had been switched (old `aicheck.ddai.monster` expired → new `aicheck.shddai.net`), but the Cloudflare R2 bucket `aichecker` CORS `AllowedOrigins` still listed only the old domain plus `http://localhost:5173`. Browser-side direct uploads (the presigned `PUT` to `*.r2.cloudflarestorage.com`) were blocked by CORS, so `fetch()` rejected with a network-level `Failed to fetch` before any HTTP response — distinct from the already-handled 504/300s analyze timeout.
- Confirmed it was a domain/CORS issue, not config: all Vercel env vars (`R2_*`, `DASHSCOPE_API_KEY`, `DATABASE_URL`, `AUTH_SECRET`, …) were present and unchanged since Mar 17; R2 object folders existed through `2026-05-28` but nothing in June, matching the breakage window right after the domain switch. The repo has no hardcoded old domain — the app uses relative `/api/...` paths and reads R2 via `R2_ENDPOINT`, and CSP `connect-src` whitelists `https://*.r2.cloudflarestorage.com` (domain-agnostic).
- Fix (done in Cloudflare console, takes effect immediately, no redeploy): updated R2 → aichecker → Settings → CORS Policy `AllowedOrigins` to `https://aicheck.shddai.net` (+ `http://localhost:5173` for local dev), keeping the existing `GET/PUT/HEAD` methods and `Content-Type` header. Uploads recovered.
- Code improvement: in `src/App.tsx`, added an `isNetworkError()` helper and replaced the raw `Failed to fetch` leaking into the UI with actionable messages — R2 `PUT` failures now point at "R2 CORS / 网络异常" (and include the HTTP status on non-network failures), and a dropped `/api/analyze` connection explains the likely timeout/CORS/network cause and notes that material is auto-saved. Verified via `npm run build`. Merged to `main` as squash commit `221dcd7` (PR #1), which auto-triggers a Vercel production deploy.

## Lesson / runbook

- When the production domain changes, update THREE places together, not just Vercel Domains:
  1. Vercel → Domains
  2. Cloudflare R2 bucket → Settings → CORS Policy `AllowedOrigins`
  3. Any other third-party callback / origin allowlists
- A raw `Failed to fetch` (vs the friendly 分析超时 message) means the browser `fetch()` rejected with no HTTP response — first suspect the direct-to-R2 `PUT` CORS, then a dropped `/api/analyze` connection. F12 → Network pinpoints which request failed and its status.
